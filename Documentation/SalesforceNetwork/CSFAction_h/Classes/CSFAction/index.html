contentFromData:fromResponse:error:

- (id)contentFromData:(NSData*)data fromResponse:(NSHTTPURLResponse*)response 
        error:(NSError**)error;
Parameters
data

NSData instance containing the HTTP response body.

response

The HTTP response to process.

error

Pointer to an error if one occurs. This parameter is guaranteed to be a valid pointer.

Return Value

The data to populate outputContent with, otherwise `nil` if nothing is appropriate.

Discussion

Subclassable method that processes the information coming from the HTTP response in order to consume it within the action response.

The default implementation uses NSJSONSerialization to consume the data as JSON data. Subclasses may override this if an action needs to perform some other custom logic.

context

@property (nonatomic,
    weak) id context;
Discussion

Object reference providing context to which network actions are similar. This can be used to assign a common value to a group of actions that should be cancelled together.

For example, a view controller can be assigned to this property to keep track of all actions scheduled from user actions performed within it. When the controller is dismissed, the pending actions invoked by that view controller may be cancelled, preventing unnecessary network overhead.

See

cancelAllActionsWithContext:]

downloadLocation

@property (nonatomic,
    strong,
    readonly) NSURL *downloadLocation;
Discussion

If the content was downloaded to a temporary file on disk, this property represents the file URL to that temporary content.

WARNING:

The file this URL points to will be deleted after the action is complete, so you must move the file to another location on disk within the completion handlers of this action if you want to persist this content.

requireBackgroundSession

Indicates if this request should be run on a NSURLSession capable of performing background uploads or downloads. 

@property (nonatomic) BOOL requireBackgroundSession ;
Discussion

Most actions transfer JSON or other informational data and therefore run on the default NSURLSession. However, if a request is capable of running on a background session (e.g. the request is performing a download that can be performed when the app is not running), settings this value to `YES` will utilize a background session when creating a task for this request.

Availability
Available in Mac OS X v10.10. Available in iOS v8.0.
