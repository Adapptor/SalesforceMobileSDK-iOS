CSFNetworkOutputCache Protocol Reference

Conforms to NSObject
Declared in CSFNetworkOutputCache.h

Overview

Protocol description that allows arbitrary classes to define caching behavior for network output.

– cacheOutputFromAction:completionBlock: required method

Caches an action response.

- (void)cacheOutputFromAction:(CSFAction *)action completionBlock:(void ( ^ ) ( NSError *error ))completionBlock

Parameters

action

The completed action to cache
completionBlock

Block to be invoked when the cache operation is completed, or nil.

Discussion

This initiates a cache operation, meant to store the relevant objects and persist them in some form of cache, as determined by the receiver.

– shouldCacheOutputFromAction: required method

Informs the network instance whether or not this cache instance is capable of handling the output from the supplied action.

- (BOOL)shouldCacheOutputFromAction:(CSFAction *)action

Parameters

action

The action that is being processed.

Return Value

YES if the receiver is capable of caching this action, otherwise NO if the receiver should be skipped.

Discussion

@discussion This permits multiple cache handlers to handle requests from a network, but only process the actions that can be accommodated by them. For example, some information should be stored in a SQLite database vs a flat-file on the file system, or cached as binary images on disk.
