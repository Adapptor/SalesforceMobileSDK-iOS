– customBubblingEventTypes

DEPRECATED: declare properties of type RCTBubblingEventBlock instead

- (NSArray<NSString*> *)customBubblingEventTypes

Discussion

Returns an array of names of events that can be sent by native views. This should return bubbling, directly-dispatched event types. The event name should not include a prefix such as ‘on’ or ‘top’, as this will be applied as needed. When subscribing to the event, use the ‘Captured’ suffix to indicate the captured form, or omit the suffix for the bubbling form.

Note that this method is not inherited when you subclass a view module, and you should not call [super customBubblingEventTypes] when overriding it.
