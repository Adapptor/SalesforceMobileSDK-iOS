RCTImageDataDecoder Protocol Reference

Conforms to RCTBridgeModule
Declared in RCTImageLoader.h

Overview

Provides the interface needed to register an image decoder. Image decoders are also bridge modules, so should be registered using RCT_EXPORT_MODULE().

– canDecodeImageData: required method

Indicates whether this handler is capable of decoding the specified data. Typically the handler would examine some sort of header data to determine this.

- (BOOL)canDecodeImageData:(NSData *)imageData

Declared In

RCTImageLoader.h

– decodeImageData:size:scale:resizeMode:completionHandler: required method

Decode an image from the data object. The method should call the completionHandler when the decoding operation has finished. The method should also return a cancellation block, if applicable.

- (RCTImageLoaderCancellationBlock)decodeImageData:(NSData *)imageData size:(CGSize)size scale:(CGFloat)scale resizeMode:(RCTResizeMode)resizeMode completionHandler:(RCTImageLoaderCompletionBlock)completionHandler

Discussion

If you provide a custom image decoder, you most implement scheduling yourself, to avoid decoding large amounts of images at the same time.

Declared In

RCTImageLoader.h

– decoderPriority

If more than one RCTImageDataDecoder responds YES to canDecodeImageData: then decoderPriority is used to determine which one to use. The decoder with the highest priority will be selected. Default priority is zero. If two or more valid decoders have the same priority, the selection order is undefined.

- (float)decoderPriority
